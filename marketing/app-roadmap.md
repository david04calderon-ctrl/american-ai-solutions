# Mobile App Roadmap — American AI Solutions LLC
**Objective:** Build a client-facing mobile app that makes American AI Solutions the most accessible, highest-value AI firm in Michigan — while also automating internal operations.

---

## PHASE 1 — Progressive Web App (PWA) — Build First, Build Fast
**Timeline:** 4–6 weeks | **Cost:** $0 beyond dev time

Before building a native app, convert the existing website into a PWA. This gives you:
- "Add to Home Screen" on iOS and Android — looks and feels like a native app
- Offline capability
- Push notifications
- Fast load on mobile networks
- No App Store approval required — ships instantly

**What to add to index.html:**
```html
<!-- In <head> -->
<link rel="manifest" href="/manifest.json">
<meta name="theme-color" content="#1A3A8F">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
```

**manifest.json** (create at root):
```json
{
  "name": "American AI Solutions",
  "short_name": "AmericanAI",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#0D1117",
  "theme_color": "#1A3A8F",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

**Service worker** (sw.js at root) — caches assets for offline use.

## PHASE 2 — Client Portal App (React Native / Expo)
**Timeline:** 8–12 weeks | **Stack:** Expo (React Native), Supabase backend, Stripe

This is the real product — a client-facing dashboard where every American AI Solutions client logs in to see their automations running.

### Core screens:

**1. Dashboard**
- Live status of all active automations ("Dispatch automation: running · 847 tasks completed this week")
- ROI meter: money/time saved this week vs. baseline
- Alert feed: anomalies flagged by AI that need human review

**2. Automation Library**
- All client automations, with on/off toggles
- Logs: what each automation did, when, and the result
- Edit triggers (e.g., change the email template the automation uses)

**3. Reports**
- Weekly ROI summary (auto-generated by Claude Opus 4.8 from automation logs)
- PDF download for sharing with leadership
- 30-day, 90-day, annual views

**4. Request New Automation**
- Chat interface: "I want to automate X"
- Claude interprets the request, generates a scoping document
- David gets notified → reviews → sends proposal
- This replaces the current contact form for existing clients

**5. Training Hub**
- Short video clips: how to use each automation
- Prompt engineering guides for their specific tools
- SOPs as searchable documents

### Backend (Supabase):
- Authentication (email + magic link, no password friction)
- Database: clients, automations, logs, reports
- Real-time: live automation status via Supabase Realtime
- Storage: PDF reports, training videos
- Edge functions: connect to n8n webhooks for automation status

### Why Expo:
- Single codebase → iOS + Android + web
- Over-the-air updates (no App Store re-review for minor updates)
- Huge component library (NativeBase, Tamagui)
- TypeScript native

### App Store Presence:
- Publish to both Apple App Store and Google Play once stable
- Listing: "American AI Solutions — AI Automation Dashboard"
- This alone differentiates you from every other AI consulting firm in Michigan

## PHASE 3 — AI-Powered Client Acquisition App (Internal Tool)
**Timeline:** Runs parallel to Phase 2 | **Stack:** n8n + Claude API + Supabase

Build internal automation tools that let you scale without hiring:

**Lead Intake Bot:**
- When a form is submitted → Claude generates a personalized 2-paragraph response + schedules a follow-up in HubSpot
- Sends David a Slack/SMS summary of the lead with context

**Proposal Generator:**
- Input: notes from strategy call
- Output: full SOW document (Word/PDF) with pricing, timeline, deliverables
- Claude Opus 4.8 writes it; David reviews and sends via Pandadoc
- Cuts proposal time from 90 minutes to 15 minutes

**Client Onboarding Bot:**
- Triggered when a contract is signed in Pandadoc
- Creates Linear project with all standard tasks
- Sends client onboarding email sequence (Fathom records the kickoff call automatically)
- Sets up Supabase record for their dashboard access

**Reporting Bot (weekly):**
- Pulls automation logs from n8n
- Claude generates a plain-English weekly summary for each client
- Sends via email automatically every Friday at 8am

## PHASE 4 — Productized SaaS (Year 2)
**Timeline:** 12–18 months out | **Stack:** Same as Phase 2 + Stripe billing

Convert the best-performing client automations into standalone SaaS products:

**Product 1: AutoQuality** — AI quality inspection SaaS for Tier 2/3 auto suppliers
- $499–$999/month subscription
- Connects to existing line cameras via API
- Self-serve onboarding

**Product 2: DispatchAI** — AI dispatch optimization for Michigan trucking companies
- $299–$799/month
- Integrates with common TMS platforms

**Product 3: ClinicOps** — AI scheduling + prior auth automation for Michigan medical practices
- $599–$1,499/month per location
- HIPAA-compliant, self-hosted option available

Each of these starts as a client engagement → becomes a productized service tier → becomes a SaaS.
This is how a $25K project becomes $12K/year in recurring revenue indefinitely.
